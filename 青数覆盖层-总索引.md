# 青数覆盖层总索引

## 1. 目标

这份索引文档用于把当前分支中“相对 `main` 需要刻意保留、且后续应按覆盖层回贴”的内容集中列出来，作为后续从最新 `main` 新拉分支后的总导航。

当前建议坚持的总策略：

1. 先文档化覆盖层。
2. 再从最新 `main` 新拉分支。
3. 最后按覆盖层分域回贴，而不是继续在当前脏分支上做高耦合大 merge。

这样做符合：

1. `KISS`：先把边界讲清楚，再迁移。
2. `YAGNI`：先保留已经明确要保留的域，不提前做大范围重命名或重构。
3. `SOLID`：把公共主干和业务覆盖层分开处理。
4. `DRY`：避免后续每次继续对齐 `main` 时反复人工搜差异。

## 3. 建议迁移顺序

后续从最新 `main` 新起分支后，推荐顺序如下：

1. 先回贴品牌覆盖层，确保产品名、品牌介绍、品牌图形语言和主题主色稳定。
2. 再回贴工作台 UI 与样式覆盖层，恢复双侧栏壳层、工作台结构和主题桥接。
3. 然后回贴登录认证覆盖层，恢复登录入口、认证配置、bridge、deep link 与登录态联动。
4. 再回贴内置治理链覆盖层，恢复 managed catalog、治理预览与只读边界。
5. 最后回贴唤醒与 TTS 覆盖层，恢复语音输入、唤醒、朗读与相关打包资源。

原因：

1. 品牌和 UI 更偏“外层壳”，先稳定视觉和结构，后续功能域更容易接。
2. 登录认证决定 server models、Portal bridge、登录欢迎浮层和 managed 可见性，应在治理链之前稳定。
3. 治理链和语音链更偏“中层能力”，需要建立在外层壳与认证入口稳定的前提下。

## 4. 四类红线

### 4.1 不要顺手重命名全部 `lobsterai`

这些标识很多已经带有兼容意义，例如：

1. deep link scheme
2. 用户数据目录
3. 数据库文件名
4. 更新与 Portal 链路
5. session key / task key

### 4.2 不要只迁 UI，不迁主题系统

如果只迁组件，不迁以下内容，最终会出现“结构像了，颜色和交互没完全像”的半成品状态：

1. `src/renderer/theme/css/themes.css`
2. `src/renderer/theme/tailwind/plugin.cjs`
3. `src/renderer/index.css`
4. `src/renderer/services/theme.ts`

### 4.3 不要只迁治理 UI，不迁主进程治理主干

如果只迁 renderer 侧的治理预览与 bundle 展示，而漏掉：

1. `src/main/qingshuModules/*`
2. `src/main/qingshuManaged/catalogService.ts`
3. `src/main/main.ts` 中的治理 IPC

那么界面会出现，但实际数据和治理分析不会成立。

### 4.4 不要只迁设置页，不迁语音运行时链路

语音相关最容易只迁 `Settings.tsx`，但真正缺一不可的是：

1. shared constants
2. `preload` IPC 暴露
3. `main.ts` IPC handler
4. `macSpeechService / wakeInputService / ttsRouterService / macTtsService / edgeTtsService`
5. `resources/macos-speech/*`
6. 构建与打包脚本

### 4.5 不要只迁登录入口，不迁认证主干

认证相关最容易只迁 `LoginButton.tsx` 或设置页里的认证配置，但真正缺一不可的是：

1. `src/common/auth.ts`
2. `src/main/auth/*`
3. `src/main/main.ts` 中的 `authWindow / deep link / IPC handler`
4. `src/main/preload.ts`
5. `src/renderer/services/auth.ts`
6. `src/renderer/store/slices/authSlice.ts`
7. `src/renderer/components/LoginWelcomeOverlay.tsx`

## 5. 当前总验收口径

后续从 `main` 新分支回贴完成后，至少要做这组总体验收：

1. 应用名、品牌文案、协议弹窗和关于页品牌区正确。
2. 工作台仍是当前分支的双侧栏壳层与主内容编排。
3. 登录入口、飞书扫码、bridge、欢迎浮层和登录态恢复链路正确。
4. managed agent / skill / tool 的锁定、只读、过滤和治理预览语义正确。
5. 语音输入、语音唤醒、TTS 设置页能读写配置并正确反映运行时状态。
6. `WakeActivationOverlay`、唤醒后听写、唤醒欢迎语、助手回复朗读都按当前分支口径工作。
